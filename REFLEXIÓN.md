# Reflexión sobre CI y Calidad del Software

## 1. ¿Qué tipo de error evita el CI?
Evita errores que rompen funcionalidades existentes, código que no pasa pruebas, y errores regresión. 

## 2. ¿Qué tipo de error NO evita?
No evitaría errores existentes en el código, a nivel de lógica de negocio y problemas de diseño

## 3. ¿Qué pasaría si un equipo ignora el CI?
Actualmente sería visto como una pérdida de calidad, pues detecta problemas rápidamente, evitando despliegues con errores.
